# Book Wish

책 위시리스트를 등록하고 관리하는 MVP 웹 애플리케이션입니다.

로그인한 사용자가 읽고 싶은 책을 목록으로 관리하고, 제목·장르 기준으로 필터링하며, 선택한 책 목록을 복사하거나 삭제할 수 있도록 구현했습니다.

단순 CRUD 구현에 그치지 않고, 인증 세션 유지, 비동기 데이터 동기화, 사용자 인지 부하 개선에 중점을 두고 개발했습니다.

---

## 주요 구현 포인트

### 1. Refresh Token 경쟁 상태 해결

Access Token 만료 시 여러 API 요청이 동시에 401 응답을 받을 경우, 각각 Refresh 요청을 보내면서 세션이 꼬이는 문제가 발생했습니다.

Refresh Token Rotation 방식에서는 첫 번째 Refresh 요청이 성공하면 기존 Refresh Token이 폐기되기 때문에, 동시에 실행된 다른 Refresh 요청은 실패할 수 있습니다.

이를 해결하기 위해 Axios 인터셉터 모듈 스코프에서 하나의 `refreshPromise`를 공유하도록 구현했습니다. 진행 중인 Refresh 요청이 있다면 새로운 Refresh 요청을 보내지 않고 기존 Promise를 기다리도록 처리하여, 토큰 갱신 요청이 한 번만 실행되도록 개선했습니다.

### 2. TanStack Query 기반 데이터 동기화

초기에는 도서 등록·수정 이후 테이블 데이터가 즉시 갱신되지 않는 문제가 있었습니다.

비동기 요청 완료 시점을 명확히 제어하기 위해 생성·수정·삭제 로직을 `useMutation` 기반으로 전환했습니다. 이후 `onSuccess`에서 `invalidateQueries`를 호출하여 서버 데이터 변경 이후 테이블 데이터가 자동으로 갱신되도록 개선했습니다.

### 3. 사용자 인지 부하를 줄이는 UI 개선

등록과 수정 폼이 같은 사이드 패널에서 동작하면서, 사용자가 현재 어떤 작업을 수행 중인지 명확히 인지하기 어려운 문제가 있었습니다.

작업 모드에 따라 필요한 UI만 노출되도록 조건부 렌더링을 적용했습니다. 또한 등록/수정 상태에 따라 타이틀 색상과 버튼 문구를 다르게 표시하여 사용자가 현재 작업 상태를 쉽게 이해할 수 있도록 개선했습니다.

## 트러블슈팅

### 1. 비동기 작업 후 데이터 갱신 누락

**문제**

도서 등록 또는 수정 이후 테이블 데이터가 즉시 갱신되지 않는 문제가 있었습니다.

**원인**

비동기 요청이 완료되기 전에 쿼리 무효화가 실행되거나, 요청 성공 이후 데이터 갱신 흐름이 명확히 분리되어 있지 않았습니다.

**해결**

도서 생성·수정·삭제 로직을 TanStack Query의 `useMutation` 기반으로 전환했습니다. 이후 `onSuccess` 콜백에서 `invalidateQueries`를 호출하여 서버 작업이 성공한 뒤 테이블 데이터가 다시 조회되도록 개선했습니다.

---

### 2. 동시 401 에러와 Refresh Token 경쟁 상태

**문제**

여러 API 요청이 동시에 401 응답을 받을 경우, 각 요청이 개별적으로 Refresh Token 요청을 보내면서 세션이 만료되거나 요청 흐름이 꼬이는 문제가 발생했습니다.

**원인**

Refresh Token Rotation 방식에서는 Refresh 요청이 성공하면 기존 Refresh Token이 폐기됩니다. 따라서 여러 Refresh 요청이 동시에 발생하면, 첫 번째 요청 이후 나머지 요청은 이미 폐기된 토큰으로 재요청하게 됩니다.

**해결**

Axios 인터셉터 내부에서 `refreshPromise`를 모듈 스코프 변수로 관리했습니다. 이미 Refresh 요청이 진행 중이라면 새로운 Refresh 요청을 보내지 않고 기존 Promise를 기다리도록 처리했습니다.

이를 통해 Refresh 요청을 한 번만 수행하고, 토큰 갱신이 완료된 뒤 실패했던 요청을 재시도하도록 개선했습니다.

---

### 3. 사용자 인지 부하를 줄이는 UI/UX 개선

**문제**

등록과 수정 기능이 같은 사이드 패널에서 동작하면서 사용자가 현재 작업 상태를 직관적으로 파악하기 어려웠습니다.

**원인**

등록/수정 모드의 UI 차이가 크지 않아 사용자가 현재 어떤 작업을 하고 있는지 추가적인 학습이 필요했습니다.

**해결**

작업 모드에 따라 필요한 UI 요소만 노출되도록 조건부 렌더링을 적용했습니다. 또한 사이드 패널의 타이틀 색상과 버튼 문구를 작업 상태에 맞게 변경하여 등록/수정 흐름을 더 명확하게 구분했습니다.
