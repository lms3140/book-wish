# Book Wish

읽고 싶은 책을 찜하고, 구매한 책의 독서 상태와 감상을 관리하는 웹 애플리케이션입니다.

찜 도서 관리에서 끝나지 않고 구매한 책을 소장 도서로 전환한 뒤, 독서 상태와 장르별 통계를 확인할 수 있도록 구성했습니다. 인증 세션 유지, 서버 데이터 동기화, 반응형 테이블, 오류 복구 UI처럼 실제 사용 중 발생할 수 있는 문제를 다루는 데 중점을 두었습니다.

---

## 주요 화면

| 경로 | 화면 | 제공 기능 |
| --- | --- | --- |
| `/login` | 로그인 | 쿠키 기반 로그인 및 인증 오류 표시 |
| `/` | 찜 도서 | 등록·수정·삭제, 필터, 정렬, 다중 선택, 복사, 구매 처리 |
| `/owned` | 소장 도서 | 등록·수정·삭제, 독서 상태·구매일·짧은 평 관리 |
| `/chart` | 통계 | 전체 소장 도서 수, 독서 상태 및 장르별 통계 |

인증이 필요한 경로에 접근하면 `/auth/me`를 통해 세션을 확인하며, 인증되지 않은 사용자는 로그인 화면으로 이동합니다.

## 주요 기능

### 찜 도서 관리

- 도서 등록, 상세 조회, 수정, 단일·다중 삭제
- 제목과 장르를 기준으로 실시간 필터링
- 제목 정렬, 페이지 이동, 표시할 열 선택
- 체크박스로 선택한 도서 정보 클립보드 복사
- 선택한 찜 도서를 일괄 구매 처리
- 등록·수정·상세 조회를 하나의 사이드 패널에서 제공

### 소장 도서 및 독서 기록

- 소장 도서 직접 등록 및 찜 도서 구매 처리 연동
- 독서 상태 관리
  - 소장 중
  - 읽는 중
  - 완독
  - 읽기 보류됨
  - 읽던 중 중단
- 구매일, ISBN, 짧은 평 기록
- 제목·장르 필터, 정렬, 열 표시 설정, 다중 삭제

### 통계

- 등록한 전체 소장 도서 수 표시
- 독서 상태별 도서 수 원형 차트
- 장르별 보유 도서 수 원형·막대 차트 전환
- 데이터가 없거나 조회에 실패한 경우 별도 상태 표시

### 사용자 경험

- 모바일 폭에서도 줄바꿈되는 반응형 테이블 도구 모음
- 등록·수정 모드에 따른 사이드 패널 강조와 버튼 문구 변경
- 선택된 행과 현재 편집 중인 행 시각적 구분
- 목록 조회 실패 시 원인을 안내하고 직접 다시 시도할 수 있는 오류 화면
- Toast 기반 작업 성공·실패 피드백
- 로컬 스토리지에 유지되는 라이트·다크 테마
- 파괴적인 작업 전 확인 다이얼로그 제공

## 최근 개선 사항

- 찜 도서와 소장 도서 테이블 도구 모음을 반응형 레이아웃으로 개선
- 페이지 이동 버튼과 주요 테이블 문구를 한국어로 통일
- 목록 조회 오류 화면에 재시도 버튼과 진행 상태 추가
- 공통 입력 필드와 테마 토글을 `customUi` 영역으로 이동해 컴포넌트 책임 정리
- 버튼 variant 정의를 `button.tsx`로 통합
- 테마 토글을 스위치 UI로 변경하고 전역 버튼에 포인터 커서 적용
- 달력 컴포넌트의 버튼 variant import와 코드 포맷 정리
- 사용하지 않던 별도 도서 필터 컴포넌트를 제거하고 테이블 내부 상태와 함께 관리

## 주요 구현 포인트

### Refresh Token 경쟁 상태 제어

Access Token이 만료된 상태에서 여러 요청이 동시에 `401` 응답을 받으면 Refresh 요청 역시 여러 번 발생할 수 있습니다. Refresh Token Rotation 환경에서는 첫 갱신 후 기존 토큰이 폐기되므로 나머지 요청이 실패할 가능성이 있습니다.

Axios 인터셉터 모듈에서 하나의 `refreshPromise`를 공유하여 이미 토큰 갱신이 진행 중일 때는 추가 요청을 만들지 않고 같은 Promise를 기다리도록 구현했습니다. 갱신이 끝나면 실패했던 요청을 한 번만 재시도하고, 갱신 자체가 실패하면 인증 상태를 해제합니다.

### TanStack Query 기반 서버 상태 동기화

도서 생성·수정·삭제·구매 처리를 `useMutation`으로 관리합니다. 서버 작업이 성공한 뒤 관련 Query Key를 무효화하여 목록을 다시 조회하므로 클라이언트 상태를 수동으로 맞추지 않아도 최신 데이터가 화면에 반영됩니다.

목록 조회에 실패하면 오류 안내와 재시도 버튼을 표시합니다. 재요청 중에는 버튼을 비활성화하고 아이콘과 문구를 변경해 현재 상태를 알 수 있도록 했습니다.

### TanStack Table과 React Compiler 호환성 대응

테이블 객체는 내부 상태가 바뀌어도 같은 참조를 유지합니다. 필터 UI를 별도 컴포넌트로 분리해 `table` 객체만 전달했을 때 React Compiler가 내부 상태 변경을 렌더링 의존성으로 추적하지 못해 입력과 열 표시 설정이 갱신되지 않는 문제가 있었습니다.

`"use no memo"`로 우회하는 대신 필터와 열 설정 UI를 테이블 컴포넌트 내부에 유지했습니다. 반복 코드 감소보다 상태 변경의 예측 가능성과 현재 컴파일러 최적화를 우선한 결정입니다. 자세한 기록은 [UI 개선 개발일지](./docs/%5B개발일지%20-%2020260722%5D%20ui%20개선%202.md)에서 확인할 수 있습니다.

### 폼과 UI 상태 분리

React Hook Form과 Zod로 입력 상태와 검증 규칙을 관리하고, Zustand는 선택된 도서·사이드 패널 모드·인증·테마처럼 화면 전반에서 공유하는 상태에 사용합니다. 서버에서 가져온 데이터는 TanStack Query가 담당하도록 역할을 구분했습니다.

## 시작하기

### 요구 사항

- Node.js `20.19+` 또는 `22.12+`
- npm
- Book Wish 백엔드 API 서버

이 저장소는 프론트엔드 애플리케이션입니다. 로그인과 도서 데이터 기능을 사용하려면 별도의 백엔드 서버가 실행 중이어야 합니다.

### 설치 및 실행

```bash
npm ci
```

프로젝트 루트에 `.env` 파일을 만들고 API 주소를 설정합니다.

```env
VITE_API_BASE_URL=http://your-api-server
```

개발 서버를 실행합니다.

```bash
npm run dev
```

API 요청에는 `withCredentials: true`가 적용되어 있습니다. 프론트엔드와 API의 출처가 다르다면 백엔드 CORS 설정에서 프론트엔드 출처와 Credential 요청을 허용해야 합니다.

### 스크립트

| 명령어 | 설명 |
| --- | --- |
| `npm run dev` | Vite 개발 서버 실행 |
| `npm run build` | TypeScript 검사 후 프로덕션 빌드 생성 |
| `npm run lint` | 전체 소스 ESLint 검사 |
| `npm run preview` | 빌드 결과 로컬 미리보기 |

```txt
src/

    customUi/          # 프로젝트 요구사항에 맞춘 조합형 UI
    header/            # 화면 내비게이션과 상단 액션
    layout/            # 페이지 및 폼 레이아웃
    ui/                # shadcn/ui 기반 기본 컴포넌트
  feature/
    Auth/              # 로그인과 인증 API
    Chart/             # 소장 도서 통계 카드와 차트
    ownedBook/         # 소장 도서 목록, 폼, API, 스키마
    wishBook/          # 찜 도서 목록, 폼, API, 사이드 패널
    bookType.ts        # 공통 도서 도메인 타입
  lib/
    api.ts             # Axios 인스턴스와 토큰 갱신 인터셉터
    toast.ts           # 공통 Toast 헬퍼
    utils.ts           # 스타일 클래스 결합 유틸리티
  router/              # 라우트, 인증 가드, 로딩·오류 화면
```

각 도서 기능은 API, 폼, 모델과 화면 컴포넌트를 기능 단위로 가까이 배치합니다. 재사용 가능한 기본 UI와 도메인 로직이 포함된 컴포넌트를 분리해 변경 범위를 명확히 했습니다.

## 데이터 처리 흐름

1. 라우터 진입 시 `/auth/me`로 쿠키 기반 세션을 확인합니다.
2. 각 페이지는 TanStack Query로 찜 도서·소장 도서·통계 데이터를 조회합니다.
3. 폼 제출과 삭제·구매 작업은 Mutation으로 서버에 반영합니다.
4. 성공한 Mutation은 관련 Query를 무효화해 최신 데이터를 다시 가져옵니다.
5. 인증 만료로 `401`이 발생하면 Axios 인터셉터가 토큰 갱신 후 원래 요청을 재시도합니다.

## 트러블슈팅 요약

### 비동기 작업 이후 목록이 갱신되지 않음

서버 요청 완료 전에 목록 갱신이 실행되던 흐름을 `useMutation`의 `onSuccess`와 `invalidateQueries` 조합으로 변경했습니다. 서버 작업의 성공 시점과 다시 조회하는 시점을 명확히 분리했습니다.

### 동시 401 응답으로 Refresh 요청이 중복됨

모든 실패 요청이 각자 토큰 갱신을 시작하지 않도록 모듈 스코프의 `refreshPromise`를 공유했습니다. 동시에 실패한 요청은 하나의 갱신 결과를 기다린 뒤 순서대로 재시도합니다.

### 등록과 수정 상태를 구분하기 어려움

사이드 패널의 모드에 따라 제목 색상, 버튼 문구와 노출할 액션을 변경했습니다. 현재 작업 중인 테이블 행에도 모드별 강조 스타일을 적용했습니다.

### 테이블 필터 분리 후 입력이 갱신되지 않음

TanStack Table 객체의 안정적인 참조와 React Compiler의 메모이제이션이 맞물려 분리된 UI가 다시 렌더링되지 않았습니다. 테이블 상태에 강하게 결합된 UI를 원래 컴포넌트 내부로 되돌려 동작의 일관성을 확보했습니다.

## 배포

Vercel SPA 배포를 위한 rewrite 설정이 포함되어 있습니다. 모든 경로를 `index.html`로 전달하므로 브라우저에서 하위 경로를 직접 열거나 새로고침해도 React Router가 화면을 처리합니다.

배포 환경에도 `VITE_API_BASE_URL` 환경 변수를 등록해야 합니다.
