# 기능확장

## 기능 확장 이유

원래 요청했던 친구가 원했던 모습은 찜목록과 복사가능한 형태만을 원했다.

나는 친구가 이 앱에 더 자주 들어오기를 원했고
책을 좋아하는 친구이니 찜목록에 더해 책을 관리할 수 있는 방향으로 발전시키면 어떨까 싶었다.

## 뭘 추가해볼까?

현재 모습은
간단한 책 필터와 crud 기능, 클립보드 기능. 끝.

여기에 나는

1. 찜목록에서 소유전환
2. 소유한 책 한줄평
3. 소유한 책의 독서 진행도
4. 독서를 포기했다면 왜 포기했는지 추적
5. 이 모든것을 유의미한 정보로 재가공

이러한 기능을 넣어보고싶었다. 내가 좋아하는 책의 장르나, 일부분이지만 습관도 체크할 수 있을것이다.

## 기대되는것?

이러한 변경으로 이 앱이 좀 더 매력적으로 바뀔거라 생각한다.
그리고 더 복잡한 앱이 되니 신경써야할것도 많아지겠지.

# 회원가입 구현

## 구현 이유

원래 회원가입은 넣지않으려고 했는데, 다른사람도 써봤으면 하는 바램으로 추가했다.

## 구현사항

회원가입 컴포넌트를 추가했다. (src/feature/Register.tsx)

id와 password, nickname 만 입력하면 회원가입이 된다. (추후 이메일이나 소셜로그인으로 대체해볼 예정)

## 한계

개발 편의성을 위해 password확인절차라던지 다 건너뛰었기때문에, 이걸 당장 쓸수는 없을거다.
본인인증 절차가 없다.

## 회원가입 구현 이슈

회원가입을 구현하다 발생한 이슈.

### 상황

회원가입을 만들고 회원가입을 완료하는것 까진 완료되었으나, 로그인이 되지않는 문제를 발견하였다.
로그인 요청은 성공적이나 refresh토큰에서 401이 발생하였다.

### 원인

원인은 samesite=none과 localhost의 문제였다.
이 앱은 현재 토큰보관을 쿠키에 하고있는데, 배포를 위해 samesite=none을 서버에 설정해놓은점이 이 이슈의 원인이었다.
samesite=none을 할 경우 https가 아니면 쿠키를 받지못한다.

### 조치

서버에서 dev환경일 경우 samesite를 Lax로 적용되도록 교체하였다.

### 결과

로컬에서 쿠키를 정상적으로 받아오게 되었다.

# Book 분리

기존 Book을 ownedBook과 wishBook으로 분리했다.

## 왜 분리했는가?

찜과 소유도서는 책이라는 공통점이 있지만 다른점이 꽤 많았다.

두 항목은 서로 가져야할 속성과 상태가 달랐고, 하는일이 다르기때문에 분리하는 쪽이 더 관리하기 쉽다고 판단했다.

그리고 처음 찜목록만 생각하고 개발했기때문에 재사용도 힘들었다.
억지로 재사용성을 챙기려니 복잡해질 가능성이 꽤 높았고, 역할이 다르다는 점 이라는 이유도 있었다.

## 구현

### 페이지 분리

WishBookPage와 OwnedBookPage를 만들어 서로 다른 페이지로 만들었다.
WishBookPage는 현재 "/" 로 매핑되어있고
OwnedBookPage는 "/owned" 로 매핑되었다.

홈을 추가하는 시점에 다시 바꿔볼 예정이다.

### WishBookPage 구매 버튼 추가

구매버튼을 추가했다.
책을 체크하고 구매 버튼을 누르게 되면 wishbook의 데이터가 "WANTED" -> "OWNED" 로 바뀌게 되어 WishBook에 뜨지않게된다. (쿼리를 그렇게 짰다.)

그리고 구매 테이블에 추가되며, 구매한 책을 관리할 수 있게된다!

## 아쉬운점

ui가 거의 비슷한 파일이 두개로 만들어졌다. 초기 계획과 많이 달라졌기에 어쩔수 없고, 하는 역할이 다르고 추후 다른 기능이 추가될경우 유리한 측면도 있다고 생각한다.
