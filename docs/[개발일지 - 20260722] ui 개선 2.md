# 이슈

1. table 필터 분리 이슈

## table 필터 분리

테이블 상단의 검색필터와 열 설정 ui가 반복되는점과 독립적인 역할을 한다는점을 들어 분리를 시도했습니다.

## 이슈발생

table 객체를 props로 전달하고, 해당 ui를 분리하는데 까진 성공했습니다.
하지만 분리된 필터와 열 설정이 제대로 동작하지 않는 일이 발생했습니다.
필터는 아무런 값을 입력해도 값이 입력되지않았고, 열 설정또한 작동하지 않았습니다.

## 왜 이런일이?

https://github.com/TanStack/table/issues/6117
table 객체는 내부 상태가 변경돼도 동일한 참조를 유지합니다. React Compiler가 이 내부 상태 변화를 렌더링 의존성으로 추적하지 못해, 이전 렌더링 결과를 재사용한 것으로 추정했습니다. 유사한 현상이 TanStack Table 이슈 #6117에도 보고되어 있어 React Compiler와의 호환성 문제로 판단했습니다.

## 해결방법

"use no memo"로 우회할 수 있었지만, 컴파일러 최적화를 비활성화하면서까지 분리할 실익이 크지 않다고 판단했습니다. 해당 UI는 테이블 상태와 결합도가 높으므로 기존 컴포넌트 내부에 유지했습니다.

## 한마디

이렇게 확실하게 이유를 알수없는 버그가 제 코드에도 발생한다는점이 꽤 놀라웠습니다.
라이브러리와 컴파일러의 충돌이라는점이 생소하기도 합니다.
